# Metodología de pruebas

1. **Carga**: Incremento hasta la carga objetivo.
2. **Estrés**: Más allá de lo esperado para forzar el fallo controlado.
3. **Spike**: Picos súbitos para probar elasticidad.
4. **Soak**: Larga duración para detectar fugas y degradación.

Recolectar métricas: p50/p95, RPS, errores, CPU, RAM, I/O, GC, conexiones DB.
