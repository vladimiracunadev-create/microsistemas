# Metodologia de pruebas

1. **Carga**: Incremento hasta la carga objetivo.
2. **Estres**: Mas alla de lo esperado para forzar el fallo controlado.
3. **Spike**: Picos subitos para probar elasticidad.
4. **Soak**: Larga duracion para detectar fugas y degradacion.

Recolectar metricas: p50/p95, RPS, errores, CPU, RAM, I/O, GC, conexiones DB.
